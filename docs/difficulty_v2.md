# difficulty v2 规则与卡池清单

## 1. 目标

- 只保留一个难度标签：`easy / medium / hard`。
- 简单模式使用 easy 池。
- 中等模式使用 easy + medium 池。
- 困难模式使用全卡池。

## 2. 热度数据来源与清洗

- 热度来源：公开可访问的 HSGuru 标准环境套牌数据聚合。
- 取样窗口：`past_day`，`rank=all`，`min_games=50`，`limit=200`。
- 先从 deck 样本中聚合卡牌加权入选率。
- 再使用本地正式标准题库进行反向清洗，只保留：
  - 当前标准环境
  - `collectible = true`
  - 正式主卡记录
  - 非奇闻组件 / 非衍生组件 / 非占位镜像
- 因此像布洛克斯加的奇闻衍生组件不会进入最终热度修正。

## 3. 具体计算规则

最终难度分：

```text
difficultyScoreV2 = difficultyBaseScore - popularityAdjust
```

其中：

### 3.1 结构基础分 difficultyBaseScore（越高越难）
- 传说、史诗、英雄牌、多职业、关键词多、种族/派系明显、提示标签多 → 降低难度
- 普通卡、中立常规随从、无关键词、无种族/派系、迷你包、超低费、提示标签少 → 提高难度
- 新标准版本会有轻微“更容易被记住”的减分

### 3.2 热度修正 popularityAdjust（越高越容易）
- 以清洗后的 HSGuru deck 聚合加权入选率为基础。
- 先做归一化：

```text
normalizedPopularity = cardInclusion / maxInclusion
```

- 再计算主修正项：

```text
popularityAdjust = normalizedPopularity * 4.2
```

- 额外热度奖励：
  - deck hits >= 20：额外 -0.3 难度
  - inclusion >= 10%：额外 -0.4 难度
  - inclusion >= 5%：额外 -0.2 难度

### 3.3 分档方式
- 全部标准卡按照 `difficultyScoreV2` 从易到难排序。
- 前 260 张：easy
- 接下来 300 张：medium
- 剩余 835 张：hard
- 中等模式实际使用池：easy + medium = 560 张

## 4. 当前结果概览

- easy：260 张
- medium：300 张
- hard：835 张
- 中等模式可用总池：560 张

## 5. 说明

- 这一版是 `v2`：结构规则 + 清洗热度联合打分。
- 热度权重已明显提高，不再只是轻微修正。
- 但仍然会有少量“经典高认知、当前热度不够”的牌没有进入更低难度；后续建议加入少量人工覆盖。
- 例如像火球术这类牌，说明只靠标准环境即时热度仍不足以完全替代人工常识。
